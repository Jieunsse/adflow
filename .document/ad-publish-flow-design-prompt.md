# Claude Design 프롬프트 — 광고 집행 플로우 개선 (목표 선택 · 영상 소재 · 특별 카테고리 · 상태 확인)

> 이 문서는 **Claude Design에 그대로 붙여 넣는 프롬프트**다. 디자인시스템(이미 생성됨)과 함께 이 문서를 참조시켜, AdFlow `광고 만들기` 2번째 스텝(② 광고 집행)의 화면을 개선 디자인한다.
> 상세 기획·결정 사항은 `.document/PRD-ad-publish-flow.md` 참조. 전체 IA·테마·컴포넌트 매핑은 `docs/PRD.md` 참조.

## 0. 한 줄 브리프

AdFlow `광고 만들기` 스텝퍼의 **② 광고 집행** 화면을 개선 디자인해줘. 이미 만들어 둔 AdFlow 디자인시스템(다크/라이트 두 테마, 기본 라이트, Pretendard, 사이드바 셸)을 그대로 따르고, 아래에 정의된 신규/변경 UI를 기존 집행 설정 화면에 자연스럽게 통합해줘.

---

## 1. 맥락

- 이 화면은 사이드바 셸 **안** `광고 만들기` 경로의 **3스텝 스텝퍼 2번째 스텝**이다.
  - ① 소재 만들기 → **② 광고 집행** ← 여기 → ③ 성과 확인(집행 성공 요약)
- 대상 사용자: **기업 내 마케터** (Meta 광고 경험 있음, 목표·포맷·카테고리 개념 이해함)
- 기존 ② 화면에서 **추가/변경**되는 요소가 있고, 기존 항목은 그대로 유지된다.

**기존 ② 화면 항목 (유지):**
광고 소재(선택 헤드라인 미니카드 + 이미지 업로드) / 랜딩 페이지 URL / 일일 예산 / 집행 기간 / 타겟 설정(연령·성별·국가·게재 위치 LOCKED) / 게재 상태(ACTIVE/PAUSED)

**이번에 추가/변경되는 요소:**
1. 캠페인 목표 선택 (신규)
2. 광고 소재 포맷 선택 (신규 — 기존 이미지 업로드를 이미지/영상 선택으로 확장)
3. 영상 업로드 + 썸네일 업로드 (신규)
4. 특별 광고 카테고리 선택 (신규)
5. 랜딩 URL 조건부 필수 처리 (변경)
6. 집행 성공 카드 "상태 확인" 버튼 (변경)

## 2. 디자인할 화면 × 상태 매트릭스

### A. ② 광고 집행 — 설정 화면 (좌측 패널)

| # | 상태 | 보여줄 것 |
|---|---|---|
| A1 | **기본 (TRAFFIC + 이미지, 기입 전)** | 전체 설정 폼. 신규 항목(목표 선택·포맷 선택·특별 카테고리) 위치와 기존 항목 배치를 명확히 보여주는 레퍼런스 컷. |
| A2 | **목표 = AWARENESS 선택 시** | 랜딩 URL 필드가 "선택 사항"으로 바뀌고, URL 없어도 집행 버튼 활성 상태. AWARENESS임을 알 수 있는 상태 표시. |
| A3 | **포맷 = 영상 선택 + 영상 업로드 완료 + 썸네일 없음** | 영상 미리보기(썸네일 자리에 영상 아이콘 또는 첫 프레임) + "썸네일 업로드(선택)" 버튼 + "미입력 시 자동 추출" 안내 캡션. |
| A4 | **포맷 = 영상 + 영상 처리 중 (Meta 업로드 후 인코딩 대기)** | 영상 업로드는 완료됐지만 Meta 처리 중 — 스피너 또는 프로그레스 + "영상 처리 중, 잠시만 기다려주세요" 안내. 집행 버튼 비활성. |
| A5 | **특별 광고 카테고리 선택됨 (예: CREDIT)** | 카테고리 선택 상태 + 바로 아래 타겟팅 제한 안내 (예: "⚠ 18세 미만 타겟 불가, 일부 국가 타겟팅 제한") 인라인 표시. |
| A6 | **집행 버튼 활성 — 모든 필수 항목 채워진 상태** | TRAFFIC + 이미지, URL·예산·기간·국가 입력 완료 → "Meta에 광고 게재하기" 버튼 활성. |
| A7 | **집행 버튼 비활성 케이스** | TRAFFIC 목표인데 URL 미입력 → 버튼 비활성 + 인라인 안내. 또는 영상 처리 중 → 비활성 + 안내. |

### B. ② 광고 집행 — 상태 패널 (우측 패널)

기존 상태 패널 구조(대기 / 전송 중 / 성공 / 에러) 유지. 변경은 성공 카드 1곳만.

| # | 상태 | 보여줄 것 |
|---|---|---|
| B1 | **집행 성공 — 상태 확인 전** | 기존 `CampaignSuccessCard` (✓ + "광고가 Meta에 성공적으로 등록됐어요" + Campaign/AdSet/Ad ID + 복사 버튼) + 신규: **"광고 상태 확인"** 버튼 (보조 버튼, 아이콘 포함) + "Meta 검토에 최대 24시간 소요될 수 있어요" 캡션. |
| B2 | **상태 확인 클릭 → 로딩** | "상태 확인" 버튼 자리에 스피너 또는 버튼 로딩 상태. |
| B3 | **상태: PENDING_REVIEW (검토 중)** | "검토 중" 배지(중립 색) + "Meta가 광고를 검토하고 있어요. 보통 몇 분~24시간 소요돼요." |
| B4 | **상태: ACTIVE (게재 중)** | "게재 중" 배지(초록) + "광고가 정상적으로 게재되고 있어요." |
| B5 | **상태: DISAPPROVED (반려)** | "문제 있음" 배지(빨강) + "Meta가 이 광고를 반려했어요." + "Meta 광고 관리자에서 사유 확인 →" 외부 링크 버튼. |

### C. (참고) 집행 요약 카드 변경 반영

| # | 상태 | 보여줄 것 |
|---|---|---|
| C1 | **집행 요약 카드 (우측 하단, 기존)** | 기존 4행(캠페인 목표 / 구매 유형 / 게재 위치 / 특별 카테고리) 중 — **캠페인 목표 행**: 선택한 목표(`트래픽` 또는 `인지도`)로 동적 표시. **특별 카테고리 행**: 선택 없으면 "없음", 선택 시 해당 카테고리명. |

## 3. 신규 UI 컴포넌트 상세

### 3.1 캠페인 목표 선택

- 위치: 집행 설정 폼의 **최상단** (가장 먼저 결정하는 항목)
- 형태: SegmentedControl 또는 RadioCard 2개 — "트래픽" / "인지도"
- 각 옵션에 짧은 설명 부연: 트래픽 = "웹사이트 방문 유도" / 인지도 = "브랜드 노출 극대화"
- 기본 선택: 트래픽
- OUTCOME_SALES는 "곧 지원" disabled 항목으로 표시 (있으면 좋음 — 선택)

### 3.2 광고 소재 포맷 선택

- 위치: 기존 "광고 소재" 섹션 안, 이미지 업로드 위
- 형태: SegmentedControl 또는 칩 2개 — "이미지" / "영상"
- 이미지 선택: 기존 이미지 업로드 UI 그대로 표시
- 영상 선택: 영상 업로드 UI + 썸네일 업로드 UI 표시 (이미지 업로드 대체)

### 3.3 영상 업로드

- 업로드 영역: 드래그앤드롭 또는 클릭 업로드 (MP4·MOV, 최대 500MB)
- 업로드 완료 시: 파일명 + 용량 + "제거" 버튼
- 처리 중: 스피너 + "Meta 서버에서 영상을 처리 중이에요…"
- 처리 완료: 체크 아이콘 + "영상 준비 완료"

### 3.4 썸네일 업로드

- 영상 업로드 바로 아래 위치
- 레이블: "썸네일 이미지 (선택)"
- 안내 캡션: "미입력 시 첫 프레임이 자동으로 사용돼요"
- 업로드 영역: 이미지 업로드와 동일한 패턴, 더 작게

### 3.5 특별 광고 카테고리

- 위치: 타겟 설정 섹션 아래 또는 독립 섹션
- 레이블: "특별 광고 카테고리 (해당 시 선택)"
- 형태: 다중 선택 칩 4개 — "신용·금융" / "채용·구인" / "부동산·임대" / "정치·선거"
- 기본: 선택 없음
- 선택 시: 칩 활성 + 바로 아래 인라인 안내 표시: "⚠ [선택한 카테고리] 광고는 18세 미만 타겟 및 일부 국가 타겟팅이 제한돼요."
- 안내 캡션(항상 표시): "해당 없으면 선택하지 않아도 돼요. 미신고 시 Meta 계정에 제재가 있을 수 있어요."

## 4. 제약 (가드레일)

- **데스크톱 기준만** (모바일 반응형 범위 밖)
- **다크/라이트 두 테마** 모두, 기본 라이트, WCAG AA 대비 이상
- 기존 ② 화면의 모든 항목(랜딩 URL·예산·기간·타겟팅·게재 상태)은 **빠짐없이 유지**
- 신규 항목이 추가되더라도 **폼 전체 흐름이 자연스럽게 이어져야** 한다 (항목 순서 제안 환영)
- "곧 지원" / "연동 후 활성화" 등 placeholder는 비활성 그대로
- 기존 "AI 느낌" 연출(보라 glow / 펄스 링 / pop-in)은 쓰지 말 것
